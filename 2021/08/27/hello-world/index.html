配置 NexT

在 Hexo 中有两份主要的配置文件，其名称一样。为了描述方便，在以下说明中，分别用：

站点配置文件 /blog/_config.yml
主题配置文件 /blog/themes/next/_config.yml

区分两个配置文件

Site 设置

打开站点配置文件，找到 # Site，按照以下说明修改并保存：

title: 标题
subtitle: 副标题
description: 简介
keywords: 关键词
author: 作者
language: 语言，zh-CN是简体中文，en是英语
timezone: 时区，默认本地时区

  • 效果图

path: 博客索引页面的根路，默认值：””
per_page: 每页显示多少篇博客文章
order_by: 文章排序，默认按日期降序

Scheme 设置

打开主题配置文件，找到 # Scheme Settings，将你需用启用的 scheme 前面注释 # 去除即可：

scheme: 设置外观，Muse 默认值，是 NexT 最初的版本，黑白主调，大量留白，Mist Muse 的紧凑版，整洁有序的单栏外观，Pisces 双栏 Scheme，小家碧玉似的清新，Gemini 与 Pisces 类似，灰白分明，轮廓清晰

darkmode 是否启用黑暗模式，默认false 是不启用，true 是启用

display: 设置侧边栏显示的时机，默认 post 是在文章页面（拥有目录列表）时显示，always 是在所有页面中都显示，hide 是在所有页面中都隐藏（可以手动展开），remove 是完全移除

文章目录设置

打开站点配置文件，找到 toc，按照以下说明修改并保存：

enable: 是否启用侧边栏文章目录，默认 true 是启用，false 是不启用
number: 设置是否自动将列表编号，默认 true 是自动编号，false 是不自动编号
wrap: 设置是否换行，默认 false 是不换行，超过长度显示为…，true 是超过长度自动换行
expand_all: 设置是否显示帖子中的所有级别，默认 false 是不显示全部级别，只显示激活部分，true 是将显示帖子中的所有级别
max_depth: 设置目录的最大深度

序言文本设置

打开站点配置文件，找到 # Post Settings，按照以下说明修改并保存：

item_text: 是否在序言中显示描述文本，默认 true 是显示，false 是不显示
created_at: 是否在序言中显示帖子创建日期，默认 true 是显示，false 是不显示
updated_at: 的 enable: 是否在序言中显示帖子更新日期，默认 true 是显示，false 是不显示，another_day: 如果更新/编辑的日期与创建日期不同，是否还显示编辑的日期，默认 true 是显示，false 是不显示
categories: 是否在序言中显示分类，默认 false 是不显示全部级别，只显示激活部分，true 是将显示帖子中的所有级别

separated_meta: 是否在序言和脚部部分中显示本文字数和阅读时长，默认 true 是显示，false 是不显示
item_text_post: 是否在序言中显示本文字数和阅读时长的文本描述，默认 true 是显示，false 是不显示
item_text_total: 的 enable: 是否在脚部部分中显示本文字数和阅读时长的文本描述，默认 false 是不显示，true 是显示
